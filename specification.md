# Helios Flow - アプリケーション技術仕様書

## 1. プロジェクト概要

本アプリケーションは、複雑な工事のシステムフローをWeb UI上で可視化し、関係者間で「次に何をすべきか」を自己解決できるようにすることを目的とした静的Webアプリケーションです。属人化しがちな業務フローの確認・質問をUIで解消し、業務効率を向上させます。

## 2. システムアーキテクチャ・制約事項

* **実行環境・ホスティング**: GitHub Pages
* **サーバーサイド処理**: 一切なし（完全静的アプリケーション）
* **データソース**: `public/data.json`（読み取り専用・Single Source of Truth）
* **状態の永続化**: ブラウザの Local Storage のみ（お気に入り機能等に活用）

## 3. 技術スタック

* **コアフレームワーク**: React 18
* **言語**: TypeScript
* **ビルドツール**: Vite
* **スタイリング**: Tailwind CSS
* **ルーティング**: React Router (`react-router-dom`)
  * ※GitHub Pages上での404エラーを回避するため、`HashRouter`を採用しています。
* **フロー図描画エンジン**: React Flow (`@xyflow/react`)
* **アイコン**: Lucide React (`lucide-react`)
* **データエクスポート**: SheetJS (`xlsx`) （Excel出力用）

## 4. 主要機能要件

### 4.1. フロー図表示とインタラクション
* **ノードとエッジの描画**: React Flow を利用し、データに基づいたフロー全体をノードとエッジ（線）で可視化。ズーム、パン（画面移動）に標準対応。
* **ホバー連動アクション**:
  * ノードへのマウスオーバーで、各種詳細情報（担当、説明、必要な入力/出力、関連リンク等）を含む「説明モーダル」を表示。
  * 現在のノードから次のノードへ向かうエッジ（線）を動的にハイライト表示。

### 4.2. 検索・フィルタリング・データ一覧
* 全データを対象としたクライアントサイドのテキスト検索、およびカテゴリ・担当者等の属性によるフィルタリング処理。
* 「フロー図」以外に「一覧表示」タブなどを備え、対象のノードを一覧から探し出してフロー上の位置へズーム移動するなどの連携機能を提供。

### 4.3. お気に入り機能
* 各ノードをお気に入り登録でき、お気に入りのみで絞り込み表示する機能。
* 登録状態は Local Storage を用いてブラウザに保存・復元される仕組みを取る。

### 4.4. その他の機能
* Excel等へのリスト出力・エクスポート機能 (`xlsx` ライブラリ)。顧客名やオーナー名の表示規則（敬称の付与等）に沿った一貫性のあるデータ整形・出力。

## 5. UI/UX デザイン方針

* 情報を俯瞰できる**フロー図中心**のUI。
* マウスオーバー（ホバー）といった少ない手順で必要な詳細データ（マニュアルへのリンク、担当ロール、入力出力情報）へアクセスできるように設計。
* 大規模なノード数にも耐えられるよう、自動レイアウトだけでなく座標の手動指定（`data.json`の`position`オブジェクト）を正とするアプローチを採用。

## 6. 主要ディレクトリ・ファイル構成

```text
heliosflow/
├── public/                 // 静的アセット
│   └── data.json           // 本システムの根幹となるフロー・ノードデータ情報
├── src/
│   ├── components/         // 再利用可能なUIコンポーネント群 (Modal等)
│   ├── hooks/              // 状態管理やロジック分離のためのカスタムフック
│   ├── pages/              // ルーティング先のページコンポーネント (Flow, List等)
│   ├── utils/              // 共通のユーティリティ関数
│   ├── App.tsx             // アプリケーションのメインエントリー・レイアウト基盤
│   ├── main.tsx            // ReactDOMへのマウント処理・プロバイダ定義
│   ├── index.css           // Tailwindのエントリーポイント
│   ├── constants.ts        // アプリケーション全体で利用される定数定義
│   └── types.ts            // TypeScriptの型定義ファイル
├── package.json            // 依存関係・スクリプト管理
├── vite.config.ts          // Viteのビルド設定
└── README.md               // プロジェクトの基本情報
```
